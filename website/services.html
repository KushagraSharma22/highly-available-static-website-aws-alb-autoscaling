Services in Use

Every service, one purpose each

Thirteen AWS services work together across networking, compute, storage, and monitoring to keep this project highly available.

Amazon EC2

Resizable virtual servers that host the application tier, launched and managed by the Auto Scaling Group.

Amazon VPC

An isolated virtual network with public and private subnets across two Availability Zones.

Application Load Balancer

Layer 7 load balancing that inspects HTTP requests and routes them to healthy targets only.

Auto Scaling

Automatically adjusts the number of EC2 instances based on CPU, request count, or custom metrics.

Amazon RDS

Managed Multi-AZ relational database with automated backups, patching, and failover.

Amazon S3

Durable object storage for static assets, backups, and logs at 99.999999999% durability.

CloudWatch

Centralized metrics, logs, and alarms that trigger scaling actions and alert the team.

SNS

Publish/subscribe notifications that deliver CloudWatch alarms to email and SMS instantly.

Route 53

Highly available DNS with health checks, latency-based and failover routing policies.

IAM

Fine-grained roles and policies that grant each service only the permissions it needs.

Security Groups

Stateful virtual firewalls controlling inbound and outbound traffic at the instance level.

Internet Gateway

Allows resources in public subnets, like the load balancer, to communicate with the internet.

NAT Gateway

Lets private-subnet instances reach the internet for updates without being publicly reachable.

CKCloudWithKushagra

A hands-on demonstration of a highly available, auto-scaling 3-tier web application built on AWS.
